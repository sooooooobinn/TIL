# 어노테이션 정리

- ``AllArgsConstructor`` : 전체 필드가 들어갈 수 있도록 전체 생성자를 생성해주는 어노테이션
- ``@NoArgsConstructor`` : 기본 생성자를 생성해주는 어노테이션
- ``@Builder`` : ``@NoArgsConstructor``에 유연함을 넣어 필요한 필드에만 값을 넣어줄 수 있도록 하는 어노테이션
- ``column`` : 엔티티의 클래스의 필드를 데이터베이스 테이블의 컬럼에 매핑
    - ``column(nullable = false)`` : 여기서 ``nullable = false``은 db에서 값을 가질 수 있는지를 검사하는 것이다. 즉, ``nullable = true``는 값이 없어도 저장이 가능하다는 것이고 ``nullable = false``는 값이 없으면 저장이 불가능하다.

    - ``column(unique = true)`` : 여기서 ``unique``는 중복 가능성을 정한다. ``unique = true``라면 중복이 불가능하다는 것이고 ``unique = false``라면 중복을 허용한다는 것이다. 


> **생성자**란? 객체를 만들 때 호출되는 특수한 메서드이다
>
> ex) public User(String email, String password)
>
> **필드**란? 클래스 안에 선언된 변수

- ``@RequiredArgsConstructor`` : service에서 final을 붙인 필드에 자동으로 생성자를 주입해준다.

- ``@Builder``: Lombok이 만들어 주는 코드로, 복잡한 생성자 대신에 간단한 체이닝 방식으로 객체를 안전하고 읽기 쉽게 만들어 주는 어노테이션이다.

- ``@EntityListeners`` : 엔터티의 생명주기 이벤트를 감지해서 특정 로직을 실행해 주는 리스너를 등록하는 어노테이션

- ``GeneratedValue`` : 엔터티의 PK 값을 DB 또는 JPA가 자동으로 생성하게 하는 어노테이션
    - ``@GeneratedValue(strategy = GenerationType.XXX)`` : ``@GeneratedValue``에는 전략이 있는데 다음과 같은 형식으로 사용한다.
        - IDENTITY(제일 많이 씀, SQL 기본) : DB가 직접 PK를 생성, INSERT 할 때 id 생성
        - SEQUENCE(Oracle, PostgreSQL) : DB 시퀀스 객체 사용, INSERT 전에 id를 미리 가져옴, 성능 최적화 가능
        - AUTO(기본값) : JPA가 DB에 맞게 자동 선택
        - TABLE(거의 사용하지 않음) : 키 관리용 테이블을 따로 둠, 성능이 좋지 않아 잘 쓰지 않음

- ``@CreatedDate``, ``@LastModifiedDate`` : 엔티티가 언제 생성됐는지/언제 수정됐는지를 자동으로 채워주는 JPA Auditing 어노테이션<br>각각 처음 저장된 시간/마지막으로 수정된 시간을 저장한다.

- ``@Enumerated`` : 자바 enum 타입을 DB 컬럼에 어떤 형태로 저장할지 정하는 JPA 어노테이션<br>``@Enumerated(EnumType.XXX)`` 형태로 타입을 지정한다.
    - ORDINAL(실무에서 거의 안씀) : enum 순서(index)로 저장, 하지만 순서를 바꾸는 순간 기존 데이터가 전부 깨진다
    - STRING : enum 이름 그대로 저장
