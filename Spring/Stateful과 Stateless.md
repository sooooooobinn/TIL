# Stateful이란?

Stateful(상태유지)는 클라이언트와 서버 관계에서 서버가 클라이언트의 상태를 보존함을 의미한다.


서버에서 클라이언트가 이전 단계에서 제공한 값을 저장하고 다음 단계에서도 저장한 상태이다.


이러한 정보들은 일반적으로 브라우저의 쿠키에 저장되거나 세션 메모리에 저장되어 상태를 유지하게 된다.

## Stateful의 문제점
stateful은 서버에서 클라이언트의 상태 정보를 저장하고 있지만 해당 서버가 멈추거나 여러 이유로 해당 서버를 못 쓰게 되어 다른 서버를 사용해야 할 경우 문제점이 발생한다.


새로운 서버에서는 기존 서버에서 가지고 있던 상태값들을 가지고 있지 않기 때문이다.

> 따라서 협업을 할 시에는 클라이언트의 상태를 따로 캐시 서버에 저장하여 사용한다.

# Stateless(무상태)
Stateless는 Stateful과 반대로 클라이언트와 서버 관계에서 서버가 클라이언트의 상태를 보존하지 않는 것을 의미한다.


즉, 클라이언트와 서버간의 통신에 필요한 모든 상태 정보들은 클라이언트에서 가지고 있다가 서버와 통신할 때에 데이터를 실어 보내는 것이 무상태 구조이다.

- 서버는 단순히 요청이 오면 응답을 보내는 역할만 수행하기 때문에 상태 유지에 대한 부하가 현저히 줄어들게 된다.
- 대량의 트래픽 발생 시에도 서버 확장을 통해 대처를 수월하게 할 수 있다.

## Stateless 문제점
상대적으로 Stateful보다 더 많은 데이터가 소모된다.

하지만 로그인처럼 유저가 로그인하고 있다는 상태를 유지해야 하는 서비스는 상태를 유지하지 않으면 로그인이 풀려버린다.

# Stateless와 토큰
로그인 유지와 같은 상태는 stateful한 상태를 사용해야 하는데, 이렇게 될 경우 서버에 부하가 생기게 된다.


그리하여 stateless 특징을 유지하면서도 로그인 상태 유지를 가능하게 하는 기술이 바로 **JWT 토큰**이다.
