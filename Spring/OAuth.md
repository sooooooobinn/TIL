# OAuth란?
Google이나 Kakao, FaceBook 등의 소셜 계정을 기반으로 간편히 회원가입 및 로그인을 할 수 있는 프로토콜이다.

### OAuth의 장단점
**장점**
- 로그인 개발/유지보수 부담 감소<br>비밀번호 저장, 해싱, 변경, 분실 처리 로직이 없어도 됨
>**해싱이란?**<br>
>임의의 길이를 가진 데이터를 고정된 길이의 데이터(해시값)로 변환하는 일방향 암호화 기술이자, 데이터를 효율적으로 저장 및 검색하는 알고리즘 이다.
- 신뢰성 있는 사용자 정보 제공<br>구글, 네이버 등에서 직접 검증된 정보(email 등)를 제공
- 소셜 연동 쉽게 구현 가능<br>로그인 + 프로필 불러오기 + 인증 절차까지 한 번에 가능
- 사용자 진입 장벽 낮음<br>사용자는 회원가입 없이 "구글로 로그인"으로 빠르게 진입
- 보안적으로 우수함<br>인증/인가를 제공업체에게 위임(우리가 직접 비밀번호를 다루지 않음)
- 다양한 플랫폼과 연동<br>하나의 인증 방식으로 웹, 모바일, 외부 API까지 모두 대응 가능

**단점**
- 추가 정보가 부족함<br>기본 프로필 정보만 제공되므로 닉네임/약관 동의 등은 별도처리 필요
- 외부 서비스 의존도 증가<br>구글/네이버 API 장애 시 로그인 불가 가능성 있음
- 프로바이더별 파싱 방식이 다름<br>구글, 네이버, 카카오 등 API 응답 포맷이 다 달라서 provider별 분기 필요
>**provider란?**
>프레임워크에서 데이터를 관리하고, 이 상태를 하위 컴포넌트나 모듈에 효율ㅈㄱ으로 공유(주입)하는 디자인 패턴 및 도구
- 추가적인 보안 검토 필요
- 동일 이메일로 다른 소셜 로그인 가능성<br>providerId로 구분 필요
- 개념이 많고 복잡하다<br>OAuth2 흐름, Security 필터 체인, 토큰 처리까지 전반적으로 학습 난이도가 높다.
