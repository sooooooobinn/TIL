# val과 var
var과 val은 Kotiln에서 변수를 사용할 때 사용하는 것이다.

## val
val은 불변 타입 변수로 한 번 값을 지정하면 값을 변경하는 것이 불가능하다.

``val a = 1`` 이런식으로 선언이 가능하고 초기화를 한다면 더 이상 수정이 불가능하다.

> kotiln은 기본적으로 타입추론을 제공하고 있기 때문에 ``val a = 1``과 같이 선언을 하여도 int형으로 알아서 들어간다.<br>
``println(a::class)`` 다음과 같이 a의 자료형을 확인을 해보면 int형인 것을 알 수 있다.

<br> 반면, ``val a: int``라고 자료형과 함께 선언을 하게 된다면 한 번까지는 값이 들어갈 수 있다.

코딩을 할 때 타입 추론이 되더라도 직접 명시하는 것이 좋다. 여러명이 같이 작업을 할 때에는 혼동이 올 수 있고 가독성이 떨어지기 때문에 명시해주는 것이 좋다.

## var
var은 가변 타입 변수로 초기화 한 후에 값을 변경하는 것이 가능하다.

하지만 ``var a: String = "ABC"``라고 선언 후 자료형을 바꿔 값을 넣는다면 오류가 발생하고 자료형을 변경하려면 자바와 같이 형변환을 해야한다.
>**kotlin에서 형변환을 어떻게 할까?**
<br>
``val a: Long = 34``라고 선언된 Long형 a를 b라는 변수를 하나 만들어 int형으로 강제 형변환 시킨다.
``val b: Int = a.toInt()``<br>
``val stringValue:String = "1234"``라고 문자열로 선언된 "1234"를 
``val stringToInt = Integer.parseInt(stringValue)``를 통하여 정수 1234로 강제 형변환 시킨다.

## 차이점
val과 var의 차이점은 초기화 후 값을 변경할 수 있는지 없는지로 나뉜다.
