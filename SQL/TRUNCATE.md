# TRUNCATE
테이블의 모든 행의 데이터를 삭제하는 문법

```
ex)
TRUNCATE TABLE users;
```
> users 테이블의 모든 데이터 삭제

### DELETE문과의 차이
`DELETE` WHERE절을 사용하여 특정 데이터를 선택적으로 삭제할 수 있으므로 트랜잭션 처리가 가능하다.<br>
`TRUNCATE`는 테이블에 있는 데이터를 즉시 삭제하며 롤백이 불가능하다. 삭제시 테이블 구조와 속성은 유지하기 때문에 `DELETE`문과는 구별된다.

### TRUNCATE 제약 조건
- 참조 무결성 제약 조건<br>
외래키(FK) 제약 조건이 설정된 테이블은 `TRUNCATE` 명령어를 직접 사용할 수 없음<br>`DELETE` 명령어를 통해 값을 삭제해야한다.
> 외래키 설정이 되어 있는 테이블에서 TRUNCATE문이 필요하다면
>```
>SET FOREIGN_KEY_CHECKS = 0;
>```
>를 통하여 외래키 제약 조건을 해지 후 사용해야한다.

- 권한 필요<br>
명령어를 실행하려면 DROP 권한이 존재해야한다. 일반 DELETE 권한으로는 실행할 수 없다.
